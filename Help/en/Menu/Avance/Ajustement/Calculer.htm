XYZ Adjustment: Generalities

The adjustment principle is relatively simple. The documents stores raw data as XYZ coordinates. During normal use, those coordinates are displayed directly in grids, and used as is for all graph computations. When XYZ adjustment is applied, the measured XYZ coordinates are changed through a transformation matrix, and they are not anymore used directly in all software parts.

Uses for XYZ adjustments are multiple. You can use this function to handle a screen offset, or to calibrate very precisely a probe to match another probe response. (This is analogous to creating a calibration file, but is simpler to use because of easiness to activate or deactivate the adjustment).

 

Compute conversion matrix from reference document

This function defines an adjustment matrix. This matrix is used only when the XYZ Adjusement checkbox is checked in the measures view.
 

 

This option automatically generates an XYZ transformation matrix, by comparing primary color (and white) measures in active document, with the same measures in the reference document. The matrix is computed to make primary colors and white coordinates in current document become exactly the same than corresponding coordinates in reference document.

For optimal precision, it is recommended that white is measured in both documents (the white measured with primary and secondary colors, not level 100 is gray scale). If the white measure is missing in one of the documents, the matrix computation will take only primaries into account, and the conversion matrix will give a much less precise results around white. For that reason, it is strongly recommended to keep activated the additional white measure when measuring primary and secondary colors (see general preferences for details)

Use cases:

1 – Training : The goal is to precisely calibrate a fast and sensible colorimeter, but whose precision is quite low, using a very precise colorimeter, but which is slower, less sensible, or less practical to use. Then, you can use repeatedly the fast colorimeter, and obtain a final result as reliable as if you used the most precise probe during the whole calibration session.

First, you make two primary colors measures (including white) of the projector to calibrate, with both probes places roughly at the same place. You can use simultaneous measures, with both probes side by side, or successive measures with probes exactly at the same place, as you like.

When primary colors and white measures available on both documents, check the "Reference" checkbox on the document linked to the reference colorimeter. Then, activate the other document window, the one using the probe to "train", and call Compute conversion matrix from reference document. Validate the confirmation message to create the matrix. The checkbox XYZ Adjust is checked. All document measures, present and future, are then adjusted. To see raw measures or adjusted coordinates, you just need to check or uncheck Adjust XYZ. You can then calibrate your projector with the trained meter.

The goal of training is to obtain a maximum precision with a relatively low precision probe, or a probe whose results can vary (for exemple when ambient temperature changes...). Generally, this kind of matrix will not be saved, because it offers a gain only during the calibration session.

 

2 – Screen offset : The goal is to make measures which take into account the colorimetric shifting of the screen, but facing the projector, to benefit of a higher light quantity to measure, therefore a better precision and faster measures.

First, measure primary colors and white facing screen, in a document which will become the reference document.

Then, create a second document, with the same probe, but place the probe facing the projector. You make then a second series of measure of the primary colors and white in this new document. In the first document, which made measures facing screen, check the "Reference" checkbox. Activate the second document, and call Compute conversion matrix from reference document. The new document is ready to perform measures facing the projector, the result will take the screen offset into account when the Adjust XYZ checkbox will be checked, and will display raw measures (pure projector light) when unchecked.

The screen offset being a screen (and room) characteristic, there is few chances of varing between two calibration sessions. The adjustment matrix is likely to be saved in a file to be reused later.
