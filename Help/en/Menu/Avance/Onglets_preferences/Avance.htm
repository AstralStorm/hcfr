Measure launch

In this group, you can deactivate confirmation message at the beginning of each measure, to avoid time loss. More important, you can activate the Controlled mode. When this checkbox is checked, ColorHCFR turn all simple measure commands in controlled mode measure commands, when the following conditions are respected:

- The current document is not the reference document

- There is a reference document

- Both documents are in the same mode: %gray or IRE

- Both documents are linked to two different probes.

When all those conditions are filled, the controlled mode icon appears in the status bar, indicating all further measure commandes will be in controlled mode.

The controlled mode performs simultaneous measures between a "simple" colorimeter, and a reference colorimeter, which is linked to the reference document. The difference between simultaneous measures and controlled mode measures is the following: each time primary colors are measured, the current document XYZ adjustment matrix is automatically recomputed, and is immediately applied to all measures. This mode is like a real time training for current document colorimeter, relatively to a reference colorimeter.

 

Calibration files – advanced use

In this group, the Activate calibration files on all tri-stimulus probes allow use of calibration files with all probes, even those which return XYZ values. It allows creating specific calibration files for a particular probe, which would have slightly derived during time. Or it can be used to enhance probe performance for a specific projector and a specific probe. This function is useful only when you have a reference colorimeter more precise than the probe to calibrate ! When this function is activated, all calibration file generation and use features become available for all tri-stimulus probe, like for HCFR probe.

The Do not use white in matrix calculus (R1.x) option allow computing calibration files and adjustment matrix ignoring White, ie using uniquely the three primary colors. This mode was used in previous ColorHCFR versions (1.x). One of the new main features of version 2.0 is taking white into account during matrix calculus, for a more precise result.

 

Luxmeter LX-1108

This group deals with luxmeter LX-1108 in the software. Just define COM Port to which the luxmeter is linked to use its measures. If a COM port is defined, but the luxmeter is not linked or not powered on, ColorHCFR will simply ignore it, and will not be perturbated in any way. So, you can let definitely this parameter set, either using your luxmeter or not.

Note : The luxmeter to PC cable is quite special, and you may have difficulties to find it in usuel commercial circuits. To build it by yourself, you need a female SubD9, a mono Jack 3.5, a two wire cable (no particular characteristics, just thin enough), and a 2.2KOhm resitor. Solder wires on the jack, and link jack signal pin to pin 4 on SubD9, jack ground to pin 2 on SubD9, and put the 2.2K resistor between pins 2 and 5 on SubD9.

When the luxmeter is detected, its measures appear in real time in ColorHCFR status bar. It's an easy way to check the connection activity. During measures, normal or simultaneous, in controlled mode or not, all values measured by the luxmeter are linked to colors measured by colorimeter(s). The only exception concerns free measures, which do not use luxmeter data.

The Imperial units checkbox allow displaying luxmeter values in Ft-cd instead of Lux. ColorHCFR only treats values in Lux internally, regardless luxmeter setting (which can display its measures in Lux or Ft-cd). This option does not change anything to computing or curve appearance, it affects only numeric values display.

When you have luxmeter values associated to colors, you can use this information in luminance, gamma, near black and near white graphs, depending on the selected option:

- Display curves with colorimeter luminance: values measured by the luxmeter are ignored, curves match only Y information from colorimeter.

- Prefer luxmeter value when available: For all measures having luxmeter data, this value is used to compute the curve. If no luxmeter data is availabe, the curve is drawn from Y returned by the colorimeter.

- Display two separate luminance curves: with this option, curves are doubled. The yellow curve displays Y from colorimeter, and orange curve displays the luxmeter data. The average curves are cyan for colorimeter average Y, and violet for average luxmeter value.

In all cases, when a luxmeter data is linked to a measure, the tooltips on every curve dots indicate the luxmeter value in Lux or Ft-cd, depending on the checkbox described above.

The Compute contrast and delta luma from luxmeter value allow using luxmeter data, when available, instead of colorimeter Y to compute contrast and delta luma on primary and secondary colors.

Note : The gamma value displayed in measure tab, computed from gray scale, uses luxmeter data only if two conditions are filled:

- Luminance curves are displayed from luxmeter values, or with two separate curves

- The Compute contrast and delta luma from luxmeter value is checked.

If one of those conditions is not respected, gamma computing is based only on colorimeter Y values.

Note : Using a luxmeter is not a guarantee to obtain a better calibration. This feature is experimental, and it may evolve in the future, from users suggestions.

 

Delta E.

This group defines how delta E is computed. Do not use luminance in delta E formula (R1.x) option allow using the "simple" formula, based only on color difference, ignoring all luminance reference. It was the only mode available in previous ColorHCFR versions (1.x). Taking luminance into account minimizes results when luminance is not 100% (ie white).

The Use gamma reference to compute gray scale delta E option allow using theoretical gray level gamma on each gray level, which minimizes value for low levels. Warning: using this option must be used with precautions. When luminance tends to zero (black), delta tends to zero, even if colorimetric shift is important.
