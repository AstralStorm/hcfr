HCFR Colorimeter

 

Advanced / Calibration file

 

Build using simultaneous measures

This option builds a calibration file using simultaneous measures between the probe to calibrate and a reference colorimeter.
 

 

This option builds a calibration file using simultaneous measures between the probe to calibrate and a reference colorimeter. You can create calibration files for all tri-stimulus colorimeters when the Activate calibration files on all tri-stimulus probes is checked in advanced preferences.

To use this feature, you must have a reference colorimeter supported by this software, and a HCFR colorimeter (or other tri-stimulus) to calibrate. You must have two documents opened at the same time, the document using the reference colorimeter having the "Reference" checkbox checked. You must then call this function with the non-reference document active (the one which pilot the probe to calibrate). If the active document when you call this function is the reference document, an error is issued.

Both probes facing the screen, at an equivalent distance, and without masking each other, just call this option to launch the measure of primary colors and white. When the measures are done, the calibration matrix is computed, a Signature asking a name or a pseudo appears (useful only when you publish your calibration files or transmit them to HCFR forums)

Then, the program display the additivity information and delta E between white measured by the reference colorimeter and the white computed using the new calibration file.

Additivity data are pure information. Large differences can be found on some projectors without invalidating calibration file. But the delta E between white measured by the reference colorimeter and calibration-computed white is very important. A big value invalidates calibration file. You can consider a delta E below 3 is acceptable. But normally, if white has been measured, the delta E should be exactly zero. If not, it means calibration matrix has been computed using only primary colors.

After the additivity display, the software proposes to save the calibration file. If you don't save the file at this stage, you can do it later using Save calibration file.

Check calibration file precision by measuring the whole grayscale with the two documents, using simultaneous measures, and examine the graphs to verify that differences between the two documents are low.

When you have a new calibration file for a projector for which no calibration file is furnished, please transmit it to the development team on the HCFR forums www.homecinema-fr.com. To help us control and publish it, please send us the two .chc measure files you made using simultaneous measures when building the calibration file.

Note : to « play » with this function, it is possible to calibrate the simulated sensor, and use it as a reference colorimeter. Of course, calibration files built this way are totally useless, because they are based on random values.
 

 

Signature:

 

The signature dialog:

 

 

This dialog appears when generating a calibration file using simultaneous measures, or manually using a reference colorimeter. You can type your name or pseudo, which will be saved in the author field inside the calibration file. This information will appear in the Sensor matrix tab in the sensor configuration dialog.

 

Additivity

 

The additivity information appear in a simple message box.

 

 

This information is put inside calibration file comments field, visible in the sensor configuration dialog.
